How does JavaScript works

JavaScript is a scripting language. JavaScript is used on web pages to improve user experience, user interface, page loading, and much more.

A web developer must add JavaScript code to HTML pages. When a user requests an HTML page that contains JavaScript, the script is sent to the user's browser, and the Internet browser handles it from there.The browser will then download any files that are linked to the HTML, such as images and CSS files. The CSS parser provides the CSS information. As long as the user's browser supports JavaScript, this will work.It works in conjunction with the browser's V8 engine. If it does, the JavaScript code will run automatically without any intervention or action from the user, resulting in a more seamless user experience. Additionally, it allows the user to view the entire web page.

What does JavaScript event loop Do

The event loop is one of the most important things to understand about JavaScript. This post explains it in simple terms.

JavaScript has a built-in event loop mechanism that supports concurrency. This model is not like models in other languages such as C or Java. Concurrency is used by the Javascript event loop to execute blocks of code in its call stack in such a way that users believe it is nonblocking, even though it does temporarily halt the process. The event loop has one main function: at each tick of the javascript run process, it looks at the call stack and checks if any function or activity is available; if none is available, it pushes items from the queue into it, and that operation is later run on the stack.

Difference between local storage and session storage


Local storage
  • Local storage, which has no expiration date and stores a larger amount of data on the client's computer in a key-value pair format. Data is never sent to the server and is only available through JavaScript and HTML5.
  • It's not session based,need to deleted html, js or manually
  • Only local storage is accessible to the client.
  • 5mb/10mb storage
  • Less older browsers support
  • Single domain. No sub-domains
Session storage
  • Session storage saves a larger amount of data on the client's computer for the duration of the current session, expiring the data when the tab is closed. Data is never sent to the server and is always available from the same tab on the client.
  • Its session based and working par window or tab
  • Only local storage is accessible to the client.
  • 5mb storage
  • More older browsers support
  • Single domain per browser tab
